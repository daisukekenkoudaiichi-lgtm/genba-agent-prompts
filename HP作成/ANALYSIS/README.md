# ANALYSIS ディレクトリ
 
参考サイト分析を記録するためのディレクトリです。
 
## 目的
 
- **良い実装を再利用する** → 他のサイトの優れたパターンを応用
- **デザイン判断を言語化する** → 「なぜこうしたのか」の根拠を記録
- **感覚ではなく根拠ベースで制作する** → Claude Code が判断基準を持つ
---
 
## ファイル構成
 
```
ANALYSIS/
├── README.md                          ← このファイル
├── [site-name]-technical.md           ← 技術仕様
├── [site-name]-design.md              ← デザイン仕様
├── [site-name-2]-technical.md         ← 複数サイト対応
├── [site-name-2]-design.md
└── (さらに複数サイト分対応)
```
 
## 命名ルール
 
**柔軟な命名で複数サイトに対応:**
 
```
✅ OK な命名例:
- kobayashi-technical.md + kobayashi-design.md
- tamaki-technical.md + tamaki-design.md
- ito-technical.md + ito-design.md
- parliament-member-1-technical.md + parliament-member-1-design.md
- site-a-technical.md + site-a-design.md
- [any-name]-technical.md + [any-name]-design.md
 
❌ NG な命名:
- site-1-technical.md（番号は避ける）
- technical.md だけ（-design.md のペアが必須）
```
 
**ルール:**
- `[site-name]` は参考サイトを識別できる名前を使用
- 必ず `-technical.md` と `-design.md` のペアで用意
- 複数サイト対応なので、サイト数に制限なし
